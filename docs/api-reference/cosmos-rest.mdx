# Cosmos SDK REST API Reference

The Cosmos SDK provides a comprehensive set of chain modules that form the foundation of the network's functionality. These modules operate at the application layer and serve a similar purpose to precompiled contracts in EVM networks (like `ecrecover`, `sha256`, and `ripemd160`).
They may also have elevated privileges allowing them to:

- Modify network state directly
- Execute privileged operations
- Influence network behavior
- Perform actions that regular accounts and smart contracts cannot

This allows for endless custom functionality while maintaining security through controlled access, similar to how EVM precompiles provide optimized native operations while restricting their usage to specific contexts.

## API Structure

### Core Modules
Core components managing all basic network functions.

- **[Bank](/api-reference/cosmos-rest/bank/send)** - Token transfers and balance queries
- **[Staking](/api-reference/cosmos-rest/staking/validators)** - Validator operations and delegation
- **[Auth](/api-reference/cosmos-rest/auth/accounts)** - Account management and authentication
- **[Distribution](/api-reference/cosmos-rest/distribution/params)** - Reward distribution and withdrawals

### Governance
Decentralized governance and account authorization.

- **[Gov](/api-reference/cosmos-rest/gov/submitproposal)** - Governance proposals and voting
- **[Group](/api-reference/cosmos-rest/group/creategroup)** - Group-based decision making
- **[Authz](/api-reference/cosmos-rest/authz/grant)** - Authorization grants and permissions

### EVM Support
Ethereum Virtual Machine compatibility layer.

- **[EVM](/api-reference/cosmos-rest/evm/account)** - Ethereum account management
- **[Fee Market](/api-reference/cosmos-rest/feemarket/params)** - EIP-1559 fee market operations

### Advanced Modules
Specialized functionality for advanced use cases.

- **[Feegrant](/api-reference/cosmos-rest/feegrant/grantallowance)** - Fee allowances and grants
- **[Vesting](/api-reference/cosmos-rest/vesting/createvestingaccount)** - Token vesting schedules
- **[NFT](/api-reference/cosmos-rest/nft/send)** - Non-fungible token operations

## üìñ API Conventions

### Authentication
- **Query endpoints** (GET): No authentication required
- **Transaction endpoints** (POST): Require proper transaction signing

### Response Format
All endpoints return JSON responses:

```json
{
  "result": {
    // Endpoint-specific data
  },
  "pagination": {
    "next_key": "string",
    "total": "number"
  }
}
```

### Error Handling
Standard HTTP status codes with detailed error information:

```json
{
  "code": 3,
  "message": "account not found",
  "details": []
}
```

### Query Parameters
Common parameters across endpoints:

- `height` - Query at specific block height
- `prove` - Include merkle proofs in response
- `pagination.limit` - Number of results to return
- `pagination.offset` - Number of results to skip
