## Curl Examples Explained[​](#curl-examples-explained "Direct link to Curl Examples Explained")

The curl options below might return a response where the node complains about the content type, this is because the `--data` option sets the content type to `application/x-www-form-urlencoded`. If your node does complain, manually set the header by placing `-H "Content-Type: application/json"` at the start of the call.

The examples also do not include the URL/IP & port combination which must be the last argument given to curl e.x. `127.0.0.1:8545`

<Info>
  **EIP-1559 Support**: Cosmos EVM fully supports EIP-1559 transaction types and includes methods like `eth_feeHistory` and `eth_maxPriorityFeePerGas`.

  **Consensus Differences**: Due to using Tendermint BFT consensus instead of Proof of Work/Stake:
  * Mining methods (`miner_*`) are no-ops or return static values
  * Uncle-related methods always return 0 or null
  * Block reorganizations are not possible
  * Engine API (`engine_*`) is not applicable - Tendermint handles consensus

  **Methods Not Implemented**:
  * `eth_createAccessList` (EIP-2930) - Access list transactions
  * `trace_*` methods - Advanced tracing functionality (except limited debug methods)
  * `engine_*` methods - Post-merge Engine API (uses Tendermint instead)
  * `admin_*` methods - Node administration
  * `les_*` methods - Light Ethereum Subprotocol
  * `clique_*` methods - Proof of Authority consensus

  For more details on how Cosmos EVM differs from standard Ethereum implementations, see [Differences from Standard EVMs](/docs/documentation/differences).
</Info>

### `web3_clientVersion`[​](#web3_clientversion "Direct link to web3_clientversion")

Get the web3 client version.

#### Parameters (0)[​](#parameters-0 "Direct link to Parameters (0)")

#### Result[​](#result "Direct link to Result")

```
 {"jsonrpc":"2.0","id":1,"result":"Cosmos/0.1.3+/linux/go1.18"}
```

#### Client Examples[​](#client-examples "Direct link to Client Examples")

Shell HTTP

```
curl -X POST -H "Content-Type: application/json" http://localhost:8545 --data '{"jsonrpc": "2.0", "id": 42, "method": "web3_clientVersion", "params": []}'
```

Websocket

```
wscat -c ws://localhost:8546 -x '{"jsonrpc": "2.0", "id": 1, "method": "web3_clientVersion", "params": []}'
```

Javascript Console

```
web3.clientVersion();
```

### `web3_sha3`[​](#web3_sha3 "Direct link to web3_sha3")

Returns Keccak-256 (not the standardized SHA3-256) of the given data.

#### Parameters (1)[​](#parameters-1 "Direct link to Parameters (1)")

1: input `hexutil.Bytes`

* Required: ✓ Yes

#### Result[​](#result-1 "Direct link to Result")

```
{"jsonrpc":"2.0","id":1,"result":"0x1b84adea42d5b7d192fd8a61a85b25abe0757e9a65cab1da470258914053823f"}
```

#### Client Examples[​](#client-examples-1 "Direct link to Client Examples")

Shell HTTP

```
curl -X POST -H "Content-Type: application/json" http://localhost:8545 --data '{"jsonrpc": "2.0", "id": 42, "method": "web3_sha3", "params": [<input>]}'
```

Websocket

```
wscat -c ws://localhost:8546 -x '{"jsonrpc": "2.0", "id": 1, "method": "web3_sha3", "params": [<input>]}'
```

Javascript Console

```
web3.sha3(input);
```

### `eth_sign`[​](#eth_sign "Direct link to eth_sign")

The `sign` method calculates an Ethereum specific signature with: `sign(keccak256("\x19Ethereum Signed Message:\n" + len(message) + message)))`.

By adding a prefix to the message makes the calculated signature recognizable as an Ethereum specific signature. This prevents misuse where a malicious DApp can sign arbitrary data (e.g. transaction) and use the signature to impersonate the victim.

<Warning>
  The address to sign with must be unlocked.
</Warning>

#### Parameters[​](#parameters-7 "Direct link to Parameters")

* Account Address

* Message to sign
