# Connect with the chain

Sometimes all you need is to interact with the network, without the overhead of running a full node. Below are quick ways to query and broadcast transactions using public endpoints or lightweight tooling.

## 1. Web Wallets

> **Recommended for general use and quick transactions**

1. Install [Keplr](https://www.keplr.app/) (Chrome/Firefox).
2. Select or add the Cosmos Hub (or your target chain).
3. Use the UI to send tokens, view balances, and interact with dApps.

## 2. CLI Queries via Public RPC

> **Great for scripts or CI jobs without local node**

### Download or build the chain binary

```bash
# Adjust version as needed
VERSION=v9.0.0
curl -LO "https://github.com/cosmos/gaia/releases/download/${VERSION}/gaiad_${VERSION}_linux_amd64.tar.gz"
tar -xzf gaiad_${VERSION}_linux_amd64.tar.gz
test -f gaiad && chmod +x gaiad
sudo mv gaiad /usr/local/bin/
```

### Query examples

```bash
# Query account balance
gaiad q bank balances cosmos1... --node https://devnet-1-rpc.ib.skip.build:443

# List staking validators
gaiad q staking validators --node https://devnet-1-rpc.ib.skip.build:443
```

### Broadcast a transaction

```bash
# Create and sign a simple send
gaiad tx bank send cosmos1... cosmos1... 1000uatom \
  --node https://devnet-1-rpc.ib.skip.build:443 \
  --chain-id 4321 \
  --gas auto --fees 500uatom \
  --yes
```

## 3. REST (LCD) Endpoints

> **Use for HTTP-based integrations**

Cosmos SDK chains expose a REST API at port 1317 by default. For the devnet:

```
https://devnet-1-lcd.ib.skip.build
```

Basic endpoints:

* `GET /cosmos/bank/v1beta1/balances/{address}`
* `POST /cosmos/tx/v1beta1/txs` (broadcast)

Refer to the [OpenAPI spec](https://api.cosmos.network/) for details.

## 4. EVM RPC Endpoints

> **For Ethereum-compatible tooling**

Connect your Ethereum tools to the EVM module:

```
RPC:       https://devnet-1-evmrpc.ib.skip.build
WebSocket: wss://devnet-1-evmws.ib.skip.build
Chain ID:  4321 (0x10e1)
```

Use with MetaMask, Hardhat, Foundry, or any Ethereum-compatible client.

# Run a full node

For validators, developers, or users who need full control and don't want to rely on public endpoints, you can run your own synced node.

---

## Join the Devnet

Running a full node gives you complete control, better performance, and eliminates dependency on public infrastructure.

### Prerequisites

- Minimum hardware requirements:
  - 4+ CPU cores
  - 8GB+ RAM
  - 500GB+ SSD storage
  - Reliable internet connection

### Quick Setup

1. **Install dependencies** (Ubuntu/Debian):
```bash
sudo apt update && sudo apt install -y \
  build-essential git curl wget jq lz4 liblz4-tool \
  protobuf-compiler pkg-config unzip
```

2. **Install Go 1.22+**:
```bash
GO_VERSION=1.22.4
curl -LO "https://go.dev/dl/go${GO_VERSION}.linux-amd64.tar.gz"
sudo rm -rf /usr/local/go
tar -C /usr/local -xzf "go${GO_VERSION}.linux-amd64.tar.gz"
echo 'export PATH=$PATH:/usr/local/go/bin:$HOME/go/bin' >> ~/.bashrc
source ~/.bashrc
```

3. **Build the binary**:
```bash
git clone https://github.com/cosmos/evm.git
cd evm
make install  # installs 'evmd'
```

4. **Initialize your node**:
```bash
evmd init "your-node-name" --chain-id 4321
```

5. **Get the genesis file**:
```bash
curl -s https://devnet-1-rpc.ib.skip.build/genesis | jq .result.genesis > ~/.evmd/config/genesis.json
```

6. **Configure peers** (add to `~/.evmd/config/config.toml`):
```toml
persistent_peers = "peer1@ip1:26656,peer2@ip2:26656"
seeds = "seed1@ip1:26656,seed2@ip2:26656"
```

7. **Start syncing**:
```bash
evmd start
```

### State Sync (Faster Alternative)

Instead of syncing from genesis, use state sync for faster setup:

```bash
# Configure state sync in config.toml
[statesync]
enable = true
rpc_servers = "https://devnet-1-rpc.ib.skip.build:443,https://devnet-1-rpc.ib.skip.build:443"
trust_height = 1000000
trust_hash = "TRUST_HASH_HERE"
```

### Endpoints After Setup

Once your node is running, it will expose:

```
Cosmos RPC:     http://localhost:26657
Cosmos gRPC:    http://localhost:9090
Cosmos REST:    http://localhost:1317
EVM RPC:        http://localhost:8545
EVM WebSocket:  ws://localhost:8546
```

### Monitoring

Monitor your node's sync status:

```bash
# Check if syncing
evmd status | jq .SyncInfo.catching_up

# View latest block
evmd status | jq .SyncInfo.latest_block_height
```

### Next Steps

- **Validator Setup**: If you want to become a validator, see the [validator guide](/docs/devnet/validators)
- **Configuration**: Tune your node for performance and security
- **Monitoring**: Set up alerting and metrics collection
- **Backup**: Implement backup strategies for your validator key

Your node will now be fully synced with the devnet and can be used for any purpose - from development to validation to serving your own applications.
