## Default Network Connection Info

| Endpoint / Detail | Example (local node) | Purpose |
|-------------------|----------------------|---------|
| **EVM JSON-RPC** | `https://cevm-01-evmrpc.dev.skip.build` | Ethereum-compatible tools (MetaMask, Hardhat, ethers.js) |
| **EVM RPC WS**   | `wss://cevm-01-evmws.dev.skip.build`   | Streaming real-time, selective data for applications |
| **Cosmos RPC** | `https://cevm-01-rpc.dev.skip.build` | Tendermint-level interactions (blocks, consensus state) |
| **Cosmos REST API** | `https://cevm-01-lcd.dev.skip.build` | Query Cosmos SDK modules (bank, staking, etc.) |
| **gRPC** | `cevm-01-grpc.dev.skip.build:443` | High-performance Protobuf endpoint for back-ends |
| **Cosmos Chain ID** | `cosmos_262144-1` | Identifier when signing Cosmos transactions |
| **EVM Chain ID** | `262144` | Numeric network identifier for EVM tools |
| **Native Token** | `ATOM` | Staking / gas token |
| **Denom Exponent** | `6` | Native token has 6 decimal places (`uatom`) |

## Using Remote Endpoints

The `evmd` CLI and other clients can target remote public nodes.

### Method 1 — configure `evmd``

Edit `~/.evmd/config/client.toml`:

```toml
# ~/.evmd/config/client.toml
node = "https://cevm-01-rpc.dev.skip.build"
```

All subsequent `evmd` commands use the remote node:

```sh
evmd q bank balances $(evmd keys show <wallet_name> --address)
```

### Method 2 — direct API calls

#### EVM JSON-RPC

```js
import { ethers } from "ethers";

const provider = new ethers.JsonRpcProvider("https://cevm-01-evmrpc.dev.skip.build");
const balance = await provider.getBalance("0x...");
```

#### Cosmos REST API

```sh
curl https://cevm-01-lcd.dev.skip.build/cosmos/staking/v1beta1/params
```
