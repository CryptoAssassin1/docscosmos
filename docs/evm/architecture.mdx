## System Overview

Cosmos EVM is built as a collection of Cosmos SDK modules that work together to provide comprehensive EVM functionality within a Cosmos SDK chain. The design maintains the security and consensus properties of Cosmos while enabling full Ethereum compatibility.

The system provides a comprehensive blockchain development framework through integration with the Cosmos SDK.

## Core Modules

### x/vm Module

The `x/vm` module is the core EVM implementation module (renamed from `x/evm` in the original evmOS) that provides Ethereum compatibility.

**Capabilities:**
- Provides Ethereum Virtual Machine compatibility
- Handles contract deployment and execution
- Manages EVM state transitions
- Supports Ethereum opcodes

## Parameter Management

Both the VM and FeeMarket modules maintain their own sets of parameters that govern their behavior. These parameters can be updated through governance proposals.

### VM Module Parameters
- EVM denomination
- Extra EIPs enabled
- Chain configuration (fork schedule)
- Active precompiled contracts

### FeeMarket Module Parameters
- Base fee change denominator
- Elasticity multiplier
- Min gas price
- Min gas multiplier

<Note>
The ability to update parameters through governance proposals maintains the decentralized governance model of Cosmos chains.
</Note>
