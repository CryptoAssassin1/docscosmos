# Cosmos EVM

Cosmos EVM is a plug-and-play solution that adds Ethereum Virtual Machine (EVM) compatibility and customizability to Cosmos SDK chains. Based on Ethermint, this implementation enables developers to build application-specific blockchains with the control and extensibility of the Cosmos SDK while providing native EVM support and seamless interoperability between Ethereum and Cosmos ecosystems.

<Note>
**Project Status**: Cosmos EVM is a fork of evmOS maintained by Interchain Labs and the Interchain Foundation. The project is fully open-source under the Apache 2.0 license. The codebase is undergoing changes in preparation for a new audit, with the first v1 release targeted for late Q2.
</Note>

## Project Overview

Cosmos EVM enables developers to build application-specific blockchains with the control and extensibility of the Cosmos SDK while providing native EVM support and seamless interoperability between Ethereum and Cosmos ecosystems.

**Key Characteristics:**
- **Forward-compatible** with Ethereum - can run any valid Ethereum smart contract and implement new features not yet available in standard EVM
- **Cosmos SDK native** - built as Cosmos modules with full integration
- **Modular architecture** - extensive customization options and tight integration with the broader Cosmos ecosystem
- **Apache 2.0 licensed** - fully open-source

### Smart Contract Example

Since Cosmos EVM aims to be forward-compatible with Ethereum, standard Solidity contracts should work:

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.19;

contract CosmosEVMExample {
    string public message;
    address public owner;
    uint256 public updateCount;
    
    event MessageUpdated(string newMessage, address updatedBy, uint256 timestamp);
    
    constructor(string memory _message) {
        message = _message;
        owner = msg.sender;
        updateCount = 0;
    }
    
    function updateMessage(string memory _newMessage) public {
        message = _newMessage;
        updateCount++;
        emit MessageUpdated(_newMessage, msg.sender, block.timestamp);
    }
    
    function getInfo() public view returns (
        string memory currentMessage,
        address contractOwner,
        uint256 totalUpdates
    ) {
        return (message, owner, updateCount);
    }
}
```

## Module Parameters

Both the VM and FeeMarket modules maintain their own sets of parameters:

### VM Module Parameters
- EVM denomination
- Extra EIPs enabled
- Chain configuration (fork schedule)
- Active precompiled contracts

### FeeMarket Module Parameters
- Base fee change denominator
- Elasticity multiplier
- Min gas price
- Min gas multiplier

<Note>
Parameters can be updated through governance proposals, maintaining the decentralized nature of Cosmos chains.
</Note>
